Writing Scripts

The first thing you must do is create a script using the the create command and give it a name of your choice. Next, use the edit command, passing it the name of the script you just created as an argument, this will open the text editor. Finally, click on the terminal, to bring it into focus, and use the run command, passing it the name of the script you want to run, to execute the script. The steps are outlined below.

NOTE:

There is a test script call 'test.rpl' which you can run using the run command or edit using the edit command. The script looks as follows: 

            thrust 10  // set the rockets thrust to 10
            burn 100   // set the burn time to 100 milliseconds
            
            sequence left { // define a sequence called left
              angle 180 // set rockets angle to 180
              run       // fires the rocket using the current values of thrust, burn, and angle
            }
            
            sequence up { // define a sequence called right
              angle 270
              run
            }
            
            sequence right { // define a sequence called left
              angle 0
              run
            }
            
            sequence down { // define a sequence called down
              angle 90
              run
            }
            
            repeat 5 { // loop 5 times
              do left  // call the sequence left
              do up    // call the sequence up
              do right // call the sequence right
              do down  // call the sequece down 

              repeat 5 { // inner loop
                do left
                do right
              }
            }

The test script makes use of every language construct available.

You can also use the parse command to run a script. For example, you can type 'parse repeat 5 { angle 10 run angle 90 run }' into the Rocket Terminal.

Using the Rocket Terminal

Commands:

create --- usage: create SCRIPTNAME

Creates a script named SCRIPTNAME. If a script with the given name already exists, create fails.

edit --- usage: edit SCRIPTNAME

Opens the script named SCRIPTNAME inside of the text editor for editing. If the script with the given name does not exist, edit fails.

set --- usage: set NAME VALUE

The set command takes in two arguments. The first argument is the NAME of the property you would like to set the value of. This name can be any of the following properties: "thurst" -thrust refers to how much thrust you would like the rocket to produce-, "angle" -angle refers to the angle at which you would like the rocket to thrust towards-, and "burn" -burn referes to how long you want the rocket to thrust at the given angle in milliseconds. The second argument is the VALUE you would like to give the property. This value can be any positive or negative number.

run --- usage: run [SCRIPTNAME]

The run command take in one optional argument SCRIPTNAME. If a script is provided the rocket will run the script until completion. If no script is provided the run command will tell the rocket controller to execute one sequence with the current values of the environment values "thrust", "angle", and "burn". This will result in the rocket producing the set amount of thrust ,at the set angle, for the set amount of burn (burntime). If a sequence is currently being executed the rocket controller will queue the sequence and execute it immediately after the current sequence finishes execution.

ls --- usage: ls

Prints the currently existing rocket scripts.

clear --- usage: clear

Clears the terminal buffer.

info --- usage: info

Prints information about the current state of the rocket to the terminal.

printenv --- usage: printenv

Prints the current value for thrust, angle, and burn to the terminal.

reset --- usage: reset

Reset will reset the rockets position, angle, and speed back to the default values. But will not reset the values of the environment variables "thrust", "angle", and "burn".
